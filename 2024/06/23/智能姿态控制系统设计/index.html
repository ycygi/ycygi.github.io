1系统简介

本系统的硬件部分包括ESP32微控制器，角度传感器，无刷电机，电调模块以及电平转换器。系统涵盖了从MPU6050传感器数据采样，A2212无刷电机的PID控制到系统数据的算法除颤，实现了精确的姿态控制功能。系统运行效果如下所示。

1.1硬件系统简介

利用ESP32的PWM功能结合电调模块完成对电机的变功率控制即可实现系统的姿态调整。随后利用MPU6050传感器结合I2C通信即可实现对系统姿态的监测，该传感器可以读取空间层面上的速度，加速度数据，利用算法即可将读取到的数据转化为角度数据。根据以上数据即可利用PID算法完成系统的姿态控制。

1.2算法简介

  • 角度读取：

    加速度计数据：加速度计测量的是重力在传感器三个轴上的分量。令垂直方向上的加速度分量为z，重力加速度为y，则通过反三角函数可以得到一个基于加速度的角度估计值。陀螺仪数据：陀螺仪测量的是角速度。在时间间隔内，根据速度与时间的积分即可实现角度的读取。

  • 除颤设计：

    互补滤波法也是一种常用于融合MPU6050传感器的加速度计和陀螺仪数据来计算角度的方法。将加速度计提供的角度信息和陀螺仪积分得到的角度变化信息以一定的比例进行融合。加速度计测量的角度相对较为准确，但对高频噪声敏感，响应速度较慢。而陀螺仪积分得到的角度变化信息响应速度快，但存在累积误差。假设加速度计测量得到的角度为a，陀螺仪积分得到的角度变化为g，互补滤波的融合公式可以表示为： x = n*(lx + a) + (1 - n)*g 其中，x是最终融合得到的角度，lx是上一时刻的角度，n是一个介于0和1之间的权重系数。通过调整权重系数，可以控制加速度计和陀螺仪数据在融合结果中的贡献比例。通常，在短时间内，陀螺仪数据的权重较大，以保证角度的快速响应；在长时间内，加速度计数据的权重较大，以修正陀螺仪的累积误差。

    低通滤波是一种信号处理技术，用于从信号中去除高频噪声或细节，保留低频成分。MPU6050传感器受到的外部环境的影响较大，原始数据往往包含高频噪声。低通滤波在这种情况下可以用来修正这些数据，提高信号的质量。本设计将低通滤波带宽设置为1kHz即可达到较好的滤波效果。

1.3物联网部署简介

本系统的物联网部署包括信号的采集，数据的输出，系统UI的部署设计以及PID参数的自动选择。本系统的传递函数为非线性系统，故当设定角度不同时需要与之对应的PID参数，系统利用switch节点完成不同数据的选择与输出，实现了理想的功能。部署情况如下图所示。

2代码介绍

系统程序主要由角度数据采样处理程序、PID控制程序以及MQTT通信程序组成。

2.1数据采样与处理程序

MPU6050传感器采用I2C协议进行通信。该传感器的寄存器配置如下表。

寄存器地址 描述 配置示例 功能
0x68 传感器的 I2C 地址 N/A 用于 I2C 通信，无需配置
0x3B 起始寄存器，用于读取数据 N/A 从这个地址开始读取加速度计和陀螺仪数据
0x1A 配置寄存器（Configuration Register） 0x03 设置 FIFO 的中断和数字低通滤波器的配置。0x03 表示启用数字低通滤波器，采样率为 1kHz
0x1B 陀螺仪配置寄存器（Gyro Configuration Register） 0x08 设置陀螺仪的量程
0x1C 加速度计配置寄存器（Accel Configuration Register） 0x00 设置加速度计的量程
0x6B 电源管理寄存器（Power Management Register） 0x00 激活 MPU6050
  • 数据的采样

根据以上信息，对系统进行编程。本段代码采用直接访问寄存器的形式，对传感器的各项功能进行必要的配置，并对传感器进行数据的采样。本代码主要围绕MPU-6050传感器的设置与数据读取展开。包括setup_mpu_6050_registers函数通过I2C配置传感器的寄存器，激活传感器、设置加速度计和陀螺仪量程及低通滤波器采样率；read_mpu_6050_data 函数同样基于I2C从传感器读取字符串数据以供后续使用。

  • 数据的处理

为保证传感器数据读取的准确性，需要对数据进行初始化工作，即求初始的平均值。

完成平均值的求取后，即可进行数据的互补滤波。以下代码使用互补滤波算法结合MPU-6050传感器的加速度计和陀螺仪数据来计算并更新角度。其中，ALPHA定义了滤波器的权重，lastAngle存储上一次的角度值dt表示时间间隔。函数首先读取传感器数据，然后通过计算加速度和角速度来更新角度，使用atan2函数和fmod函数确保角度值在0到360度之间。最后，每20次循环打印一次角度和其他相关数据。

2.2无刷电机的PID控制

完成角度的采样监测后即可进行设备的PID控制。本程序通过ESC类初始化电机，并设置其引脚、PWM范围和启动阈值。接着，使用PID类创建一个PID控制器对象，设定其输入输出参数和PID调节参数。在command函数中，根据设定的输出值和角度范围，调整电机的输出功率，以适应不同角度的控制需求。在checkForSerialCommands函数中，通过串口接收用户输入的PID参数，实时更新PID控制器的参数，并反馈更新结果。

2.3MQTT通信

  • MQTT发送程序

为保证数据的一致性，系统需要向云端发送数据，以便UI端的信息接收与处理。首先判断是否成功连接MQTT服务器，未连接则启动重连程序；接着判断数据通道是否畅通，若否则重新连接；然后调用内置计时程序，每隔100ms执行一次，将测量的角度数据、功率数据和设定角度数据进行格式化处理并转换为字符串，最后向对应的主题发送这些数据以保持数据一致性。

  • MQTT接收程序

该程序用于接收订阅主题的消息，并根据主题内容更新设定角度或PID参数。当接收到特定主题的消息时，它会解析消息内容，设置相应的PID参数或更新设定角度，并通过串口打印出变更的参数。同时，使用myPID.SetTunings()函数更新PID控制器的参数。
